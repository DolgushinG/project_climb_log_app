# AI Support — флоу по экранам (для бэкенда)

Краткая выжимка: что происходит на каждом экране, что можно и нельзя, как формировать ответы Support. Support — общая поддержка (регистрация, оплата и др.).

---

## Общее

- **Stateless** — история чата на клиенте, передаётся в `history` при каждом `/chat`.
- **page** + **event_id** — контекст, откуда открыт чат. Support может давать контекстные ответы.
- **suggested_actions** — кнопки под ответом. Типы: `link`, `cancel_registration`.

---

## Экран за экраном

### checkout (Оформление — индивидуальное)

**Что делает пользователь:** выбирает пакет участия, размеры мерча, вводит промокод, оплачивает (ссылка/чек).

**Типичные вопросы:** «Как оплатить?», «Не пришла ссылка», «Ошибка при оплате», «Как отменить заявку?».

**Support может:**
- Дать ссылку на оплату через `link` (url → страница оплаты).
- Предложить отменить регистрацию через `cancel_registration` (event_id известен).

**Ограничения:** `cancel_registration` срабатывает только если пользователь авторизован и заявка ещё не оплачена (API `/cancel-take-part`).

---

### group-checkout (Оформление группы)

**Что делает пользователь:** оформляет оплату за группу участников, выбирает пакеты по каждому, загружает чеки.

**Типичные вопросы:** «Как оплатить группу?», «Куда загрузить чек?», «Можно отменить участника?».

**Support может:**
- Дать ссылку на оплату (`link`).
- `cancel_registration` — отмена всей группы или отдельного участника. Клиент вызывает `cancel-take-part` для event_id (логика отмены группы — на бэкенде).

**Ограничения:** групповая отмена может отличаться от индивидуальной (бэкенд решает по event_id и типу заявки).

---

### group-register (Заявить группу)

**Что делает пользователь:** добавляет участников (новых или из «Мои спортсмены»), выбирает категории, сет, возраст. Отправляет заявку. Может добавить в лист ожидания (pending).

**Типичные вопросы:** «Как добавить участника?», «Как выбрать категорию?», «Что такое pending?».

**Support может:**
- Объяснить шаги.
- Дать `link` на инфо/регламент (если есть URL).

**Ограничения:** `cancel_registration` здесь не имеет смысла — заявка ещё не оплачена, отмена через выход с экрана / сброс черновика.

---

### event (Детали соревнования)

**Что делает пользователь:** смотрит инфо, результаты, статистику. Может нажать «Участвовать» (→ checkout) или «В лист ожидания».

**Типичные вопросы:** «Как зарегистрироваться?», «Где результаты?», «Как отменить участие?».

**Support может:**
- `link` — на оплату, регламент, результаты.
- `cancel_registration` — если пользователь уже в checkout/зарегистрирован, event_id известен.

**Ограничения:** если пользователь только просматривает (не зарегистрирован), `cancel_registration` бесполезен.

---

### main (Список соревнований)

**Что делает пользователь:** листает текущие/завершённые соревнования, фильтр по городу.

**Типичные вопросы:** «Как найти соревнование?», «Где оплатить?» (общие вопросы).

**Support может:**
- Ответить текстом, дать `link` на FAQ, календарь и т.п.

**Ограничения:** `event_id` = null. `cancel_registration` не имеет смысла — неизвестно, какое событие имеется в виду.

---

### profile (Профиль)

**Что делает пользователь:** смотрит подписку, настройки, историю участия, премиум.

**Типичные вопросы:** «Как оформить подписку?», «Как отменить подписку?», «Где мои соревнования?».

**Support может:**
- `link` — на оплату подписки, условия, поддержку.

**Ограничения:** `event_id` = null. `cancel_registration` — только если поддержка даёт ссылку на страницу события (через `link`).

---

## suggested_actions: когда что

| Действие | Где имеет смысл | Требования |
|----------|-----------------|------------|
| **link** | Везде | `url` — валидный URL. Открывается во внешнем браузере. |
| **cancel_registration** | checkout, group-checkout, event | `event_id` обязателен. Пользователь должен быть авторизован и иметь заявку на отмену. |

Для `cancel_registration` клиент вызывает `POST /api/event/{event_id}/cancel-take-part`. Бэкенд должен проверять права и тип заявки.

---

## Когда вызываются endpoints

| Endpoint | Момент вызова |
|----------|---------------|
| **GET /status** | При открытии экранов. Кэш 5 мин. `enabled: false` → кнопка не показывается. |
| **POST /chat** | При каждой отправке сообщения. sync. |
| **POST /event** | `modal_open` при открытии чата, `action_clicked` при нажатии на suggested_action, `session_end` при закрытии. |
| **POST /feedback** | При 👍/👎 под ответом. Fire-and-forget. |

---

## Формирование ответов Support

1. **page** — подстраивать ответ под экран (checkout → про оплату, group-register → про участников).
2. **event_id** — при наличии можно давать ссылки на конкретное событие и предлагать `cancel_registration`.
3. **history** — возвращать полную историю в ответе (включая новое сообщение и свой ответ).
4. **Авторизация** — опциональна. Гости не могут использовать `cancel_registration`.
