# Backend: Собственные сеты упражнений

Спецификация API для создания, сохранения и выполнения пользовательских сетов упражнений.

---

## 1. Расширение GET /api/climbing-logs/exercises

### 1.1 Поля в ответе (для подсказок как в плане)

Чтобы в собственном сете показывались те же «Польза для скалолазания» и «Как выполнять», что и при обычном плане, в каждом объекте упражнения должны быть:

| Поле | Тип | Описание |
|------|-----|----------|
| `description` или `climbing_benefit` | string? | Текст «Польза для скалолазания» |
| `hint` | string? | Текст «Как выполнять» |

Если этих полей нет, соответствующие блоки в приложении не показываются.

### 1.2 Новое поле в ответе

| Поле | Тип | Описание |
|------|-----|----------|
| `tags` | string[] | Теги для быстрого поиска (например: `["плечи", "пресс", "fingerboard"]`) |

### 1.3 Новые query-параметры

| Параметр | Описание | Пример |
|----------|----------|--------|
| `tags` | Фильтр по тегам (OR — упражнение содержит хотя бы один тег) | `?tags=плечи,пресс` |
| `search` | Поиск по имени (name, name_ru) — частичное совпадение | `?search=подтяг` |

### 1.4 Категории

- **ofp** — ОФП (общая физподготовка)
- **sfp** — СФП (специфическая физподготовка)
- **stretching** — растяжка, стрейчинг (один раздел)

Без отдельной категории `recovery` — упражнения восстановления могут иметь `category=stretching` и теги `["восстановление", "мобильность"]`.

## 2. CRUD сохранённых сетов

### 2.1 POST /api/climbing-logs/custom-exercise-sets

Создание сета (шаблона).

**Request:**
```json
{
  "name": "Утренняя разминка",
  "exercises": [
    {
      "exercise_id": "pull_ups",
      "order": 0,
      "sets": 3,
      "reps": "10",
      "hold_seconds": null,
      "rest_seconds": 90
    },
    {
      "exercise_id": "back_stretch",
      "order": 1,
      "sets": 2,
      "reps": "30",
      "hold_seconds": 30,
      "rest_seconds": 30
    }
  ]
}
```

**Response:** `201 Created`
```json
{
  "id": 1,
  "name": "Утренняя разминка",
  "exercises": [
    {
      "exercise_id": "pull_ups",
      "order": 0,
      "sets": 3,
      "reps": "10",
      "hold_seconds": null,
      "rest_seconds": 90
    },
    {
      "exercise_id": "back_stretch",
      "order": 1,
      "sets": 2,
      "reps": "30",
      "hold_seconds": 30,
      "rest_seconds": 30
    }
  ],
  "created_at": "2025-03-15T10:00:00Z",
  "updated_at": "2025-03-15T10:00:00Z"
}
```

### 2.2 GET /api/climbing-logs/custom-exercise-sets

Список сохранённых сетов пользователя.

**Response:** `200 OK`
```json
{
  "sets": [
    {
      "id": 1,
      "name": "Утренняя разминка",
      "exercises_count": 5,
      "created_at": "2025-03-15T10:00:00Z"
    }
  ]
}
```

### 2.3 GET /api/climbing-logs/custom-exercise-sets/{id}

Детали сета с полным списком упражнений.

**Response:** `200 OK`
```json
{
  "id": 1,
  "name": "Утренняя разминка",
  "exercises": [
    {
      "exercise_id": "pull_ups",
      "order": 0,
      "sets": 3,
      "reps": "10",
      "hold_seconds": null,
      "rest_seconds": 90
    }
  ],
  "created_at": "2025-03-15T10:00:00Z",
  "updated_at": "2025-03-15T10:00:00Z"
}
```

### 2.4 PUT /api/climbing-logs/custom-exercise-sets/{id}

Обновление сета. Тело как в POST.

**Response:** `200 OK` — обновлённый объект сета.

### 2.5 DELETE /api/climbing-logs/custom-exercise-sets/{id}

Удаление сета.

**Response:** `200` или `204`

## 3. Схема упражнения в сете

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| exercise_id | string | да | ID из каталога упражнений |
| order | int | нет (default 0) | Порядок в сете (для сортировки) |
| sets | int | да | Количество подходов |
| reps | string | да | Повторения («10», «30», «max») |
| hold_seconds | int \| null | нет | Секунды удержания (для растяжки) |
| rest_seconds | int | нет (default 90) | Отдых между подходами в секундах |

## 4. Интеграция с exercise-completions

Выполненные упражнения из собственного сета сохраняются через существующий эндпоинт:

**POST /api/climbing-logs/exercise-completions**

```json
{
  "date": "2025-03-15",
  "exercise_id": "pull_ups",
  "sets_done": 3,
  "weight_kg": 25.5,
  "notes": ""
}
```

История выполнений отображается в общем экране истории (ClimbingLogHistoryScreen).

## 5. Отмена выполнения (выход во время тренировки)

При нажатии «Назад» во время выполнения своего сета показывается диалог:
- «Выйти из тренировки? Ваш сет сохранён. Его можно найти в «История моих сетов». Прогресс упражнений не сохранится.»
- Кнопки: «Остаться» / «Выйти»

**Текущее поведение:** сет остаётся сохранённым на бэкенде, пользователь может найти его в истории.

**Возможное расширение (будет просить бэк):** при подтверждении «Выйти» фронтенд может вызывать бэкенд, например:
- для логирования прерванной сессии;
- для удаления только что созданного (чернового) сета, если бэк поддерживает такую логику.

Эндпоинт для отмены/прерывания — на усмотрение бэкенда.
