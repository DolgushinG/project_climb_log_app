# Бэкенд: фильтрация по возрасту и группы

## 1. GET /api/event/{id}/available-sets?dob=YYYY-MM-DD

**Реализовано на бэкенде:**
- Фильтрация по возрасту через `is_valid_year_for_event` для переданной даты рождения.
- Обязательный `dob` при `is_input_birthday`; без `dob` — пустой массив `[]`.
- 404 при отсутствии события.

**Формат ответа:**
```json
{
  "availableSets": [
    {
      "id": 1,
      "number_set": 1,
      "free": 5,
      "time": "10:00-12:00",
      "list_pending": false,
      "day_of_week": "Сб",
      "allow_years_from": 2013,
      "allow_years_to": 2014
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | ID сета |
| `number_set` | int | Номер сета |
| `free` | int | Свободных мест |
| `time` | string | Время |
| `list_pending` | boolean | true = мест нет, лист ожидания |
| `day_of_week` | string | День недели (для отображения) |
| `allow_years_from` | int \| null | Год рождения от (fallback на фронте) |
| `allow_years_to` | int \| null | Год рождения до (fallback на фронте) |

Пустой массив `[]` — ни один сет не подходит по возрасту. Фронт показывает «Ваш возраст не подходит для участия».

## 2. GET /api/event/{id}/available-category?dob=YYYY-MM-DD

**Реализовано на бэкенде:**
- Только для `is_auto_categories` = YEAR или AGE.
- Для MANUAL/RESULT — `[]`.
- Обязательный `dob`; без `dob` — `[]`.
- 404 при отсутствии события.

**Формат ответа:**
```json
{
  "availableCategory": [
    {"category": "2 группа (2014-2013)"}
  ]
}
```

Пустой массив `[]` — группа не определена. Фронт показывает «Группа не определилась».

**Вариант с выбором сета:** `?dob=YYYY-MM-DD&number_set=N` — категории для конкретного сета. Используется, когда `available-category?dob=` вернул `[]`, но пользователь выбрал сет — попытка получить категорию по сету.

---

## 3. allow_years_from / allow_years_to в number_sets

Добавлены в ответ `available-sets`. В `prepare_event` (ответ события) сеты из БД уже содержат эти поля.

Если фронт не вызывает `available-sets`, можно опираться на фильтрацию по `allow_years_from` и `allow_years_to` в `event.sets`.

---

## Примечание: двойная фильтрация на фронте

Если `available-sets` возвращает сеты, не все из которых подходят по возрасту (например, все 3 сета вместо 1), фронт дополнительно фильтрует результат по `allow_years_from` и `allow_years_to` с учётом года рождения. Это страховка на случай сбоя фильтрации на бэкенде.
