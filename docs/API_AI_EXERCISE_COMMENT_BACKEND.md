# Backend: AI-комментарий к упражнению

Когда пользователь спрашивает про упражнение у AI-тренера, ответ можно сохранить как доп. комментарий к упражнению. Это позволит показывать подсказку от AI при следующем просмотре упражнения.

## Контекст

Пользователь на экране «Выполнить упражнения» нажимает «Спросить об упражнении у AI» на карточке упражнения. Открывается чат AI-тренера с предзаполненным вопросом. AI отвечает. Бэкенд может сохранить этот ответ и привязать к `exercise_id`.

## Передача exercise_id

Клиент передаёт `exercise_id` в поле `context` при вызове:

- `POST /api/ai/chat`
- `POST /api/ai/chat/async`

**Тело запроса (дополнение):**

```json
{
  "message": "Что это за упражнение «Репитеры»? Как выполнять, типичные ошибки.",
  "context": {
    "exercise_id": "repeaters_7_13"
  },
  "history": []
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `context.exercise_id` | string | ID упражнения из каталога/плана. Примеры: `repeaters_7_13`, `max_hangs_3_5_7`, `stretch_12345` (для растяжки из плана). Передаётся только при вопросе про конкретное упражнение. |

## Сохранение ответа AI

Когда в `context` есть `exercise_id` и ответ AI успешно сгенерирован:

1. Взять текст ответа ассистента (`message.content`).
2. Сохранить как «AI-комментарий» к упражнению для данного пользователя.

### Варианты хранения

**Вариант A: отдельная таблица**

```sql
CREATE TABLE exercise_ai_comments (
  id SERIAL PRIMARY KEY,
  user_id INT NOT NULL REFERENCES users(id),
  exercise_id VARCHAR(64) NOT NULL,
  comment_text TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, exercise_id)  -- один комментарий на упражнение, обновлять при новом вопросе
);
```

**Вариант B: поле в профиле/настройках**

Если комментарии привязаны к «моим упражнениям», можно хранить JSON:  
`{ "repeaters_7_13": "текст от AI", ... }`.

### Логика

- При `status == 'completed'` и `context.exercise_id` присутствует:
  - Сохранить `result.message.content` как `comment_text` для `(user_id, exercise_id)`.
  - Upsert: если запись есть — обновить, иначе вставить.

## Отдача комментария клиенту

Чтобы клиент мог показывать сохранённый AI-комментарий рядом с упражнением:

**Вариант 1:** добавить в `GET /api/climbing-logs/exercises` (или аналог) поле `ai_comment` для каждого упражнения, если у пользователя есть сохранённый комментарий.

**Вариант 2:** отдельный endpoint `GET /api/exercise-ai-comments?exercise_ids=id1,id2,...` — возвращает `{ "id1": "текст", "id2": "текст" }`.

**Вариант 3:** включить в ответ плана дня (`GET /api/plan/calendar/day`) — для каждого упражнения в `exercises` и `stretching` добавить `ai_comment`, если есть.

## Пример flow

1. Пользователь тапает «Спросить об упражнении» на «Репитеры» (`exercise_id: repeaters_7_13`).
2. Клиент отправляет `POST /api/ai/chat/async` с `context: { "exercise_id": "repeaters_7_13" }`.
3. AI генерирует ответ.
4. Бэкенд при `completed` сохраняет ответ в `exercise_ai_comments` для `(user_id, repeaters_7_13)`.
5. При следующем открытии плана/упражнений клиент запрашивает `ai_comment` для отображаемых `exercise_id` и показывает блок «Подсказка от AI» под упражнением.

## Повторный запрос — экономия токенов

**Важно:** когда пользователь спрашивает про упражнение повторно, клиент **сначала** запрашивает сохранённый комментарий. Если он есть — показывает его сразу, **без нового запроса к AI**. Это экономит токены.

### Flow

1. Пользователь тапает «Спросить об упражнении» на «Репитеры».
2. Клиент вызывает `GET /api/ai/exercise-comments?exercise_id=repeaters_7_13`.
3. Если ответ содержит `comment` — показываем его в модальном окне. Кнопка «Спросить снова» — только тогда делаем новый запрос к AI.
4. Если комментария нет (404 или пусто) — открываем чат и отправляем запрос к AI как обычно.

### Endpoint для получения комментария

```
GET /api/ai/exercise-comments?exercise_id={exercise_id}
```

**Заголовки:** `Authorization: Bearer <token>`

**Ответ (200) — комментарий есть:**
```json
{
  "comment": "Репитеры — это упражнение на..."
}
```

**Ответ (404) или пустой `comment`** — комментария нет, клиент делает новый запрос к AI.

## Чеклист для бэкенда

- [ ] При `completed` и наличии `exercise_id` — сохранять ответ в хранилище (таблица или JSON).
- [ ] Реализовать `GET /api/ai/exercise-comments?exercise_id=X` для отдачи сохранённого комментария (экономия токенов при повторном вопросе).
- [ ] Реализовать отдачу `ai_comment` в exercises/план дня (опционально, для отображения под упражнением).
- [ ] (Опционально) Ограничить длину `comment_text` (например, 2000 символов).
