# Платформа скиллов AI-чата (auto-detect)

## Обзор

Бэкенд автоматически определяет скилл по тексту сообщения. Пользователь просто пишет — ничего выбирать не нужно.

- **Новый чат** — не передавать `skill`; бэкенд делает auto-detect по ключевым словам
- **Существующий чат** — передавать только `conversation_id`; скилл берётся из диалога
- **Ответ API** — содержит `skill_id` для отображения в UI, какой ассистент отвечает

---

## Логика на бэкенде

| Ситуация | Поведение |
|----------|-----------|
| Пользователь передал `skill` | Используется переданный скилл |
| Существующий чат | Скилл берётся из диалога |
| Новый чат, `skill` не передан | **Auto-detect** по ключевым словам; при неуверенности → `default` |

### Как работает detection

Проверяется текст сообщения (без учёта регистра):

- **training_plan**: план трениров, недел, упражнен, объём, период…
- **nutrition**: питание, еда, калори, белок, веган, до/после тренировки…
- **psychology**: психолог, тревог, страх, высот, мотивац…
- **gear_recommendation**: обувь, туфл, обвязк, гри-гри, веревк, снаряж…
- **recovery**: восстановление, растяжк, сон, массаж, травм…
- **competition_analysis**: соревнован, формат, тактик…
- **climb_analysis**: видео, разбор, техник, баланс, resting…

Нет совпадений или слабые → `default`.

---

## Что делать на фронте

### Отправка сообщений

- **Новый чат** — не передавать `skill` (body: `message` + опционально `history`)
- **Существующий чат** — передавать только `conversation_id` + `message`
- **Ответ** — `skill_id` в ответе, показать в UI (список чатов, шапка чата)

### Список чатов

- Бэкенд возвращает `skill_id` в `GET /api/ai/conversations` — показывать бейдж/подпись скилла в карточке
- `GET /api/ai/skills` — для маппинга id → имя (кэшировать при входе в AI-чат)

### Шапка чата

- `GET /api/ai/conversations/{id}/messages` возвращает `skill_id` — показать имя скилла
- В новом чате `skill_id` приходит в первом ответе

## GET /api/ai/skills

Список скиллов для маппинга id → имя в UI.

**Response 200**
```json
{
  "skills": [
    { "id": "default", "name": "Универсальный тренер" },
    { "id": "training_plan", "name": "Тренировочные планы" },
    { "id": "nutrition", "name": "Питание скалолазов" },
    { "id": "psychology", "name": "Спортивная психология" },
    { "id": "gear_recommendation", "name": "Подбор снаряжения" },
    { "id": "recovery", "name": "Восстановление" },
    { "id": "competition_analysis", "name": "Анализ соревнований" },
    { "id": "climb_analysis", "name": "Анализ восхождения" }
  ]
}
```

## Ручной выбор скилла (опционально)

Экран выбора скилла остаётся опциональным для будущего. При необходимости можно передать `skill` в body — бэкенд использует его вместо auto-detect.
