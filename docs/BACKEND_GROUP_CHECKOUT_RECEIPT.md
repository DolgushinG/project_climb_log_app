# Backend: групповой checkout — чек по участнику

## Эндпоинт

`GET /api/event/{id}/group-checkout` — в ответе каждый элемент `group[]` должен содержать флаг, есть ли уже прикреплённый чек у этого участника.

## Поля в объекте участника группы

| Поле | Тип | Описание |
|------|-----|----------|
| `has_receipt` / `receipt_attached` | `boolean` | `true` — чек прикреплён → показываем «Чек на проверке» |

## Загрузка чека

`POST /api/event/{id}/upload-receipt` (multipart) — с полем `user_id` (ID участника) для привязки чека к конкретному участнику. Уже реализовано на фронте.
