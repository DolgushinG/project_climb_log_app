# Backend: групповой checkout — чек по участнику

## Эндпоинт

`GET /api/event/{id}/group-checkout` — в ответе каждый элемент `group[]` должен содержать флаг, есть ли уже прикреплённый чек у этого участника.

## Поля в объекте участника группы

| Поле | Тип | Описание |
|------|-----|----------|
| `has_receipt` | `boolean` | предпочтительно: `true` — чек прикреплён |
| `receipt_attached` | `boolean` | альтернатива |
| `receipt_uploaded` | `boolean` | альтернатива |
| `receipt_file_id` / `receipt_id` | any | если задан — считаем чек загружен |
| `receipts` | `array` | если не пустой — чек есть |

Приложение проверяет все эти поля. При `true` / непустом значении показывается «На проверке» в карточке участника.

## Загрузка чека

`POST /api/event/{id}/upload-receipt` (multipart) — с полем `user_id` (ID участника) для привязки чека к конкретному участнику. Уже реализовано на фронте.
