# API страницы скалодрома для Flutter (мобильное приложение)

Документация описывает эндпоинт страницы скалодрома — аналог веб-страницы `/gyms/{gym}` (gym/index.blade.php).

---

## Обзор

Экран скалодрома показывает:
- **Информация о скалодроме**: название, адрес, сайт, телефон, часы работы
- **Соревнования**: соревнования, проведённые в этом скалодроме
- **Вакансии**: список вакансий с пагинацией
- **Карта**: URL для карты

**Публичный эндпоинт** — без Authorization.

---

## 1. Данные страницы скалодрома

```
GET /api/gyms/{gym_id}
```

**Query (опционально):**
- `page` — страница вакансий (по умолчанию 1)
- `per_page` — количество вакансий на страницу (по умолчанию 10, макс. 50)

**Ответ (200):** см. полный пример в основном документе API.

**Ошибки:**
- `404` — скалодром не найден

---

## 2. Поиск скалодромов

```
GET /api/search-gyms?query=поиск
```

До 10 результатов по названию или городу.

---

## 3. Реализация в приложении

- **GymProfileScreen** — экран профиля скалодрома
- **Модели**: `Gym`, `GymEvent`, `GymJob` в `lib/models/Gym.dart`
- **Сервис**: `GymService` в `lib/services/GymService.dart`
- **Навигация**: в деталях соревнования карточка скалодрома кликабельна и ведёт в профиль (по `climbing_gym_id` или через поиск по имени)
